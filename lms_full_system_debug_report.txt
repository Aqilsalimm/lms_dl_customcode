--------------------------------------------------------------------------------
1. ANALISIS KESTABILAN LMS SETTINGS (MONITORING & INTEGRASI BACKEND)
--------------------------------------------------------------------------------
Seluruh tab pengaturan pada dashboard admin telah diperiksa dan dikonfirmasi
tidak lagi bersifat statis (front-end saja). Integrasi backend berjalan lancar:

A. Tab Course:
   - Course Visibility: Terintegrasi dengan `CourseController@index` dan
     `CourseController@show`. Siswa diwajibkan login untuk melihat kursus atau
     detail kursus jika opsi diaktifkan. Pengunjung umum akan diredirect ke login.
   - Course Content Access: Mengizinkan instruktur/admin melihat konten kursus 
     tanpa pendaftaran (enrollment check dinamis pada `CourseController@learn`).
   - Content Summary: Menampilkan ringkasan kurikulum di halaman detail kursus 
     secara kondisional sesuai variabel pengaturan.
   - Spotlight Mode: Mengubah tata letak pemutar materi di `Learn.vue` secara 
     dinamis menjadi layar penuh (fullscreen) dengan menyembunyikan header/footer.
   - Auto Complete Course: Memicu pencatatan kolom `completed_at` secara otomatis
     pada database ketika seluruh sesi pembelajaran/kuis telah diselesaikan.

B. Tab Monetization:
   - WooCommerce Integration: Telah dihapus sepenuhnya sesuai permintaan untuk 
     menghindari redundansi sistem.
   - Custom eCommerce System: Berjalan penuh. Transaksi diatur langsung via model 
     `Order` dan `Enrollment`.
   - Auto Complete Orders: Toggles berfungsi. Jika aktif, status order langsung 
     ditandai 'completed' dan user langsung terdaftar ke kelas/bundle tanpa 
     menunggu webhook pembayaran.
   - Midtrans Payment Gateway: Konfigurasi default siap pakai dengan fallback 
     mock payment yang aman untuk tahap development.
   - Revenue Sharing: Dinamis menyimpan persentase bagi hasil instruktur/admin,
     deduct fees, batas minimum penarikan (minimum withdrawal), dan masa tunggu 
     ketersediaan saldo (minimum days).

C. Tab Design:
   - Course Builder Logo: Render dinamis logo kustom base64 di navbar utama 
     (`GuestLayout.vue`) dan pemutar kelas (`Learn.vue`).
   - Columns & Courses Per Page: Jumlah item paginasi dikontrol dinamis dari DB 
     di `CourseController@index` dan kelas kolom CSS disesuaikan otomatis di grid.
   - Instructor List Layout Styles: Format tata letak kartu instruktur (portrait,
     cover, minimal) dirender dinamis di `Show.vue` detail kelas.

D. Tab Notifications:
   - Backend Notification Class: Dikelola oleh `LmsNotification.php` melalui 
     sistem database notification bawaan Laravel.
   - Pemicu Otomatis: Terhubung ke Eloquent event hook (`Enrollment` created/deleted)
     untuk pendaftaran kelas, pembatalan, dan penghapusan siswa.
   - Dropdown Bell: Badge unread notifikasi interaktif di header `DashboardWrapper.vue`.
   - Simulator: Panel khusus di halaman admin settings untuk melakukan uji coba 
     pemicu (trigger) notifikasi secara real-time.

E. Tab Certificate:
   - Authenticated Access: Menyediakan endpoint `/courses/{slug}/certificate` 
     untuk mengunduh sertifikat resmi bagi siswa yang telah lulus.
   - Multi-Theme Layouts: Mendukung 3 tema kustom (Classic, Modern, Premium Gold).
   - Cetak PDF: Dilengkapi tombol cetak dan optimasi CSS `@media print` sehingga 
     dapat dicetak/disimpan langsung sebagai dokumen PDF landscape A4 yang rapi.

2. ANALISIS COURSE BUILDER ENGINE (RESOLUSI BUG TYPO STATUS CODE)
--------------------------------------------------------------------------------
Modul pembuatan materi belajar (Course Builder) diperiksa secara mendalam:
* Pengaturan Modul (Sesi/Bab): Penambahan, pembaruan, dan penghapusan berjalan stabil.
* Pengaturan Materi (Lesson): Mendukung video link (YouTube/Vimeo), embed Canva/slaid,
  serta input teks biasa.
* Pengaturan Kuis & Soal: Mendukung tipe soal Pilihan Ganda, Benar/Salah, dan Essay.
* Resolusi Bug HTTP Status Code:
  - Ditemukan bug typo di `CourseBuilderController@store` pada baris ke-80:
    [SEBELUMNYA] return response()->json([...], 21); // Status 21 tidak valid
    [SEKARANG]   return response()->json([...], 201); // Diperbaiki ke 201 Created
  - Perbaikan ini mencegah potensi error pembacaan respons pada client/browser.

3. INTEGRASI API FRONTEND & BACKEND (VUE & LARAVEL)
--------------------------------------------------------------------------------
* Komponen Inertia Shared State (`HandleInertiaRequests.php`) menyalurkan data 
  global (`settings`, `auth.notifications`, `auth.user`) dengan overhead minimal.

  - `checkout`: Memicu generate Snap Token Midtrans secara dinamis atau auto-complete order.
  - `test-trigger`: Endpoint simulator notifikasi untuk kebutuhan developer.
* Aset Vue dikonfirmasi terkompilasi sempurna (`npm run build` selesai tanpa error).

--------------------------------------------------------------------------------
4. SISTEMATIKA FLOW SYSTEM (END-TO-END WORKFLOW)
--------------------------------------------------------------------------------
Alur kerja sistem (System Flow) terbagi menjadi tiga alur terpadu:

Siswa (Student Flow):
1. Pengunjung/Siswa menelusuri daftar kelas (bisa dibatasi login via Course Visibility).
2. Membeli kelas -> memilih metode transaksi (Midtrans / Mock Payment / Auto Complete).
3. Transaksi sukses -> Otomatis terdaftar (Enrollment) & mendapat notifikasi Course Enrolled.
4. Masuk ke halaman kelas (`Learn.vue`) -> Mempelajari materi & mengisi kuis interaktif.
5. Klik "Selesai Sesi" -> Progres diperbarui real-time.
6. Lulus/Menyelesaikan semua sesi -> Pemicu modal selamat dan muncul tombol "Unduh Sertifikat".
7. Cetak Sertifikat -> Memilih salah satu tema (Classic, Modern, Premium Gold) -> Print PDF.

Instruktur (Instructor Flow):
1. Akses Dashboard Instruktur -> Ringkasan statistik siswa terdaftar.
2. Kelola Kelas -> Menggunakan Course Builder untuk menyusun silabus materi dan kuis.
3. Import Kursus -> Menggunakan fitur import file Excel/CSV untuk menambahkan banyak kelas.
4. Menambahkan instruksi rekening penarikan dana sesuai monetization settings.

Admin (Admin Flow):
1. Kontrol penuh LMS settings -> Mengonfigurasi branding, visibilitas kelas, biaya, 
   dan tema sertifikat.
2. Verifikasi/Uji Coba notifikasi melalui simulasi notifikasi.
3. Manajemen transaksi siswa dan monitoring aktivitas instruktur.
